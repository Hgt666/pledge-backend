💎 Pledge-Backend 区块链质押借贷后端服务

基于 Go + GORM + Ethereum 开发的去中心化质押借贷后端系统

该项目为完整 Web3 链上业务后端实战项目，模拟去中心化质押借贷平台核心能力，包含链上区块扫描、合约读写交互、链上数据结构化落库、资产利息计算、风控指标核算、定时离线结算等企业级核心功能，可直接作为简历核心项目、面试实战案例。




---
📌 项目简介

本项目是去中心化质押借贷（Lending & Borrowing）后端服务，对接以太坊系智能合约，实现链上质押、存款、借款、还款、清算全链路数据落地与业务计算。

区别于简单 Demo 项目，本项目采用批量扫链、事件驱动、离线计算、原子数据库更新等生产级方案，解决链上同步慢、数据不准、并发覆盖、精度丢失等常见业务问题。

核心业务场景

- 用户链上 Deposit 质押存款 → 自动落库、更新池子总存款

- 用户链上 Borrow 抵押借款 → 记录借款信息、更新池子总借款

- 用户 Repay 还款 / Withdraw 赎回 → 实时更新池子资产数据

- 后台定时离线计算：存款利息、借款利息、用户资产

- 后台风控指标计算：LTV 抵押率、账户健康度、清算判定

- 支持合约原生读写调用：读取池子信息、用户借贷数据、链上配置


---
⚙️ 技术栈

- 后端框架：Golang、Gin

- 数据库：MySQL 8.0（GORM、批量CRUD、唯一索引优化）

- 区块链 SDK：go-ethereum（abigen 合约绑定、链上事件监听、合约读写）

- 定时任务：cron 离线定时结算

- 并发优化：批量扫块、批量入库、内存缓存状态、无阻塞协程

- 数据处理：big.Int 高精度链上金额计算、杜绝浮点精度丢失


---
🔥 项目核心亮点（面试重点）

1. 生产级批量扫链架构（解决单块同步卡顿问题）

- 摒弃传统单块循环扫描，采用批量区块同步，单次处理 200+ 区块

- 断点续扫机制，记录本地最新区块高度，重启不丢数据、不重复扫块

- SafeBlock 安全块机制，避免链上区块回滚导致数据错乱

- 批量解析事件 + 批量入库，大幅降低 MySQL IO 压力，同步速度提升 500%+

2. 链上事件驱动 + 数据库原子更新

- 监听合约 Deposit/Withdraw/Borrow/Repay 核心事件，结构化解析链上日志

- 采用 数据库原子累加 更新借贷池总额，解决高并发覆盖问题

- 基于 TxHash 唯一索引实现幂等防重，杜绝重复入账

3. 离线定时业务计算（企业标准架构）

扫链只做落库与统计更新，复杂计算离线执行，保证同步服务高性能：

- 定时重算用户存款利息、借款利息（秒级时间差精准计算）

- 自动核算 LTV 抵押率、账户健康度

- 健康度阈值判断，自动标记可清算风险账户

4. 标准合约交互方案

- 使用 abigen 自动生成合约 Go 绑定代码，手写封装读写接口

- 区分读方法（无Gas）、写方法（需签名）统一封装

- 全局单例合约实例，避免重复连接 RPC、重复初始化开销

5. 规范数据库设计

- 金额统一使用 String + big.Int 存储，解决链上超大数值精度丢失问题

- 核心字段建立唯一索引、普通索引，优化查询与去重

- 软删除、时间戳、状态位标准化设计，贴合企业业务规范

🧮 核心业务公式（面试必背）

1. 存/借款利息计算

利息 = 金额 × 年化利率 × 持仓秒数 / (365×24×3600×10000)

2. LTV 抵押率

LTV = 用户借款总价值 / 质押资产总价值

3. 账户健康度

HealthRate = 清算阈值LTV / 当前用户LTV

健康度 ≤ 1 → 触发清算风险
